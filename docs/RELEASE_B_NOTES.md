## Model card

- **Name**: SleepStager
- **Version**: 0.1.0 (model-internal); 0.2.0 (app)
- **Architecture**: Tiny transformer
  - Hand-rolled multi-head self-attention (pre-norm, GELU FF), 2 layers, d_model=64, n_heads=4
  - Replaces `nn.TransformerEncoderLayer` because `coremltools` cannot lower
    `F.multi_head_attention_forward` cleanly.
- **Input**: `features` MultiArray Float32 shape `(1, 16, 9)`
  - 16 windows × 30 s = 8-minute lookback
  - 9 features: `hr_mean, hr_std, hr_slope, accel_mean, accel_std, accel_energy, event_count_like_snore, hrv_proxy, motion_count`
- **Output**: `logits` MultiArray Float32 shape `(1, 4)` over `[wake, light, deep, rem]`
- **Deployment target**: iOS 17, watchOS 10
- **Compiled-on-install**: yes (`.mlpackage` → `.mlmodelc` by Xcode at build time)

### Training data — important caveat

This release ships a model trained on **synthetic data** generated by a
realistic Markov-chain sleep simulator (`python/training/data/dataset.py`):

- 4×4 row-stochastic stage transition matrix calibrated against published
  population sleep architecture (REM-heavy late, deep-heavy early)
- Per-stage Gaussian feature distributions matching real biomarker ranges
  (e.g. wake = high HR + high accel; deep = low HR + low HRV + minimal motion;
  REM = elevated HRV + positive HR slope)
- 12-epoch training run, cross-entropy loss converges from 1.17 → 0.31

**This is not a clinical model.** Synthetic data ⇒ the model has learned the
shape we encoded, not the shape of any individual user's sleep. Real-night
accuracy will improve once we collect labeled overnight recordings.

The model is intentionally small and conservative; on novel real data it
falls back gracefully toward the prior (more "light" than "deep/REM").

### Inference fallback

If `SleepStager.mlpackage` fails to load at runtime — corrupted bundle,
incompatible OS — `SleepEngineFactory` returns the heuristic
`FallbackHeuristicStageInferenceModel` and Settings → About shows
"Heuristic (fallback)". The app stays functional.

## HealthKit write-back

On session end, if Settings → "Share with HealthKit" is enabled and Apple
Health authorization is granted for sleep analysis, the app writes one
`HKCategorySample` per contiguous stage span:

| App stage | HKCategoryValueSleepAnalysis |
|-----------|------------------------------|
| wake      | `.awake` |
| light     | `.asleepCore` |
| deep      | `.asleepDeep` |
| rem       | `.asleepREM` |

Each sample carries `HKMetadataKeyExternalUUID = sessionId` so the user can
trace samples back to a specific session in Apple Health → Sleep.

Failures are non-fatal and logged to the diagnostic store; the local DB
remains the source of truth.

## How to retrain and re-bundle

```bash
./scripts/train_and_export_model.sh
# or, to skip training and just re-export from existing checkpoint:
./scripts/train_and_export_model.sh --skip-train
```

Requires Python 3.11 (`brew install python@3.11`). The script creates a
`.venv-export/` venv on first run and installs `torch >=2.2`, `numpy >=1.26`,
`coremltools >=7.2`. Subsequent runs reuse the venv.

After the script finishes:
1. `./scripts/generate_xcode_project.sh`
2. `xcodebuild build -project apple/SleepTracker.xcodeproj -scheme SleepTracker-iOS …`
3. Confirm `SleepStager.mlmodelc` shows up inside the `.app` bundle.

## Known limitations going into TestFlight

- Synthetic-data caveat above.
- Apple Watch is sensor-only; on-device inference still happens on iPhone.
- No HRV streaming yet (HRV is approximated from short-window RMSSD; a real
  `HKAnchoredObjectQuery` over `heartRateVariabilitySDNN` is Release C work).
- Model is not personalized. Per-user calibration is on the C/D roadmap.
